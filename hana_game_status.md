# 《Hana (花)》- 社會派懸疑文字解謎遊戲 開發進度總結

📅 **更新日期**：2026-04-13

## 1. 遊戲核心與現狀
- **類型**：第一人稱視角、AI對話驅動的社會派懸疑解謎遊戲。
- **目標**：玩家需透過與好友角色的手機對話、探索實體場景，推敲出好友「小花」突然退群並封鎖所有人的真實原因。
- **最新破關流程 (Verdict)**：遊戲已捨棄了全開自動結算的舊機制。現在玩家必須手動按下「進入最終作答」，並在 AI 系統前連續回答 4 題事實核定（如：小花媽媽怎麼了？為什麼退出群組？是誰說漏嘴？）外加 1 題無厘頭的柏宇加碼題，全數過關後才會自動進入最後的結局報告與毒舌綜合評價。

## 2. 角色設定狀態

1. **花語安 (小花)**：核心謎團人物。設定為離線狀態，玩家無法與其互動。
2. **吳思妤 (Siyu)**：會計師。是引導玩家解鎖「小楷」的關鍵突破口。
3. **陳柏宇 (Boyu)**：攝影工作室老闆。具備「手機對話」與「面對面」雙重模式。已修正其 AI 邏輯，現在他「非常清楚知道自己**沒有被封鎖**（因為親自跑去堵過小花）」。
4. **小楷 (Kai)**：**動態解鎖角色**。遊戲開局隱藏，需從思妤口中套出名字後觸發解鎖。目前已實裝小楷解鎖後會主動發送「景點推薦」與「謝謝你去年幫我醍醐灌頂」的開場白，作為帶出出軌動機的暗線。
5. **(新增) 群組「陳柏宇請吃飯~」**：加入一個無回應的舊群組，展示小花最後離群的假對話情境，提升帶入感。

## 3. 系統機制與進階互動 (最近更新)

1. **相關度分類器升級 (Relevance)**
   - API 從 `high`, `mid`, `low` 判定。改寫了嚴格的分類器 Prompt：現在如果玩家胡亂猜測「有沒有私情、是不是男女朋友」，系統會直接壓為 `mid` 或 `low` 避免玩家陷入錯誤紅鯡魚；只有命中「加護病房、說漏嘴、慶功宴」等核心元素才會判定為 `high`。
2. **卡關自動提示系統 (Hint System)**
   - 記錄對話狀態，如果玩家對某一位 NPC **連續講了 5 句低相關廢話**，系統對話框會彈出「需要提示嗎？」的點擊按鈕。
   - 點擊後，系統會偵測你對這位 NPC「還欠缺哪一條核心成就/線索」，動態推播下一句引導提示（例如：試著問問看小楷是不是在推銷保險時說漏嘴了什麼？）。榨乾後會提示你去尋找別人。
3. **動態 UI 優化**
   - 介面與文字全方位等比例放大，提升手機版與閱讀的易讀性。
   - **線索/成就通知呼吸燈效果**：當玩家取得新成就或線索時，右上角的「📱 手機」按鈕與下方導覽列的「📋 線索板」會開始跳動擴大發光 (`.alert-bounce`)，引導玩家去查閱，直到點開後才會消除。
